# PRD: 영화 감상 기록 블로그

## 프로젝트 개요

| 항목 | 내용 |
|------|------|
| **프로젝트명** | 씨네로그 (CineLog) |
| **목적** | Notion을 CMS로 활용한 개인 영화 감상 기록 및 큐레이션 블로그 |
| **CMS 선택 이유** | Notion API를 활용하여 비개발자도 영화 데이터를 직접 입력·수정·관리 가능. 별도 관리자 페이지 개발 불필요 |

---

## 주요 기능

1. **영화 카드 목록**: Notion 데이터베이스에서 영화 목록을 불러와 카드 그리드로 렌더링
2. **장르 필터링**: 장르 태그 클릭으로 원하는 장르의 영화만 필터링
3. **영화 상세 페이지**: 포스터·감상평·평점·관람일 등 상세 정보 표시

## Notion 데이터베이스 구조

데이터베이스명: `영화 기록`

| 필드명 | 타입 | 설명 |
|--------|------|------|
| `title` | Title | 영화 제목 (영문/한글) |
| `poster_url` | URL | 영화 포스터 이미지 URL |
| `genre` | Multi-select | 장르 태그 (드라마, 액션, SF 등) |
| `rating` | Number | 개인 평점 (0.0 ~ 5.0) |
| `watched_at` | Date | 관람일 |
| `status` | Select | 상태 (`봤다` / `보는 중` / `보고 싶다`) |
| `review` | Rich Text | 간단한 감상평 (스포일러 없는 한줄~세줄) |
| `director` | Rich Text | 감독 이름 |
| `release_year` | Number | 개봉 연도 |
| `published` | Checkbox | 웹 노출 여부 (false면 초안으로 숨김) |

## 화면 구성

### 1. 메인 페이지 (`/`)
- 최신 감상 영화 카드 그리드 (3열, 반응형)
- 장르 필터 탭 (전체 / 드라마 / 액션 / SF / 공포 / ...)
- 각 카드: 포스터 썸네일, 제목, 평점 별표, 관람일

### 2. 영화 상세 페이지 (`/movies/[id]`)
- 포스터 이미지 (좌) + 메타 정보 (우) 레이아웃
- 제목, 감독, 개봉 연도, 장르 뱃지
- 별점 시각화 (★☆ 표현)
- 감상평 본문

### 3. 상태별 위시리스트 (`/watchlist`)
- `보고 싶다` / `보는 중` / `봤다` 탭으로 분류
- 간략한 리스트 형태 (포스터 썸네일 + 제목 + 장르)

## MVP 범위

MVP에 포함할 최소 기능:
- Notion API 연동 및 영화 데이터 fetch
- 메인 페이지 카드 그리드 렌더링
- 장르 필터링 (클라이언트 사이드)
- 영화 상세 페이지

MVP에서 제외:
- 위시리스트 페이지 (2차)
- 검색 기능 (2차)
- 댓글/반응 기능 (2차)
- 페이지네이션 (초기 50건 고정)

## 구현 단계

### 1단계 — 환경 설정 및 Notion 연동 (Day 1)
- Notion Integration 생성 및 API 키 발급
- 데이터베이스 생성 및 샘플 데이터 5건 입력
- `@notionhq/client` 설치 및 `lib/notion.ts` 작성
- 환경 변수 설정 (`NOTION_API_KEY`, `NOTION_DATABASE_ID`)

### 2단계 — 핵심 UI 구현 (Day 2)
- 영화 카드 컴포넌트 (`MovieCard`)
- 메인 페이지 그리드 레이아웃
- 장르 필터 컴포넌트
- 로딩 스켈레톤 UI

### 3단계 — 상세 페이지 및 마무리 (Day 3)
- 영화 상세 페이지 (`/movies/[id]`)
- ISR(Incremental Static Regeneration) 설정 (revalidate: 3600)
- 반응형 레이아웃 점검
- Vercel 배포
